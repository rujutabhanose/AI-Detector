# AI Slop Detector
An end-to-end implementation of an AI writing detector

https://ai-detector-9rnzye9bybttzqzgjg4crw.streamlit.app

## Data
To train our classifier, we needed many examples of human-written text and AI-generated text.

### Human-written text
Gather examples of human-written text by crawling and scraping Wikipedia.
- For each page seen, two random paragraphs were stored
- 10,001 paragraphs were stored in total
- The code for crawling and scraping is contained in `/scraping`
- The result is in `/scrapes`.

### AI-generated text
AI text was then generated by prompting a model to rewrite each paragraph from Wikipedia.
- This was done in two passes (1. summarize, 2. rewrite) with separate contexts, to accurately model real-world generation
- GPT 5 Nano was used because it is cheap, sufficient, and a member of the most popular model family
- The Batch API was used because it suited our use case
- The code for data generation is contained in `/data-generation`

## Model
The model is a LoRA adapter of `bert-base-cased`, with a classification head.
- Validation accuracy: 96%
- Training was done with HuggingFace
- The code for training is contained in `/training`


## Benchmark Performance and Limitations
The model was tested on the [RAID benchmark](https://github.com/liamdugan/raid) training set to get a sense of the model's limitations.

### RAID Benchmark Performance
The model performed better than random guessing across all model families and most domains, indicating that general, transferable properties of AI-generated writing were learned.

The best TPR@FPR=1% measures were seen for IMDB movie reviews, Wikipedia, and arXiv abstracts.
- The TPR@FPR=1% for reviews was about 90%
- The TPR@FPR=1% for Wikipedia, arXiv abstracts, and news articles was between 30% and 40%
- The TPR@FPR=1% for book excerpts, recipes, poetry, and Reddit posts was between 1% and 20%

Text generated by LLaMA was the most likely to be detected.

The code and results for this analysis is contained in `/raid`.
